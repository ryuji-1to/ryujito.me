# Design Guidelines

## Purpose

このドキュメントは、このサイトの見た目と体験を一貫させるための設計メモです。
新しいページやコンポーネントを追加するときは、ここにある原則を優先します。

## Product Tone

- 個人サイトとして静かで誠実な印象を保つ
- 情報を過剰に飾らず、内容そのものを読みやすく見せる
- プロダクト感よりも、ノートやポートフォリオに近い距離感を目指す
- 軽さよりも、落ち着きと読みやすさを優先する

## Design Principles

### 1. Content First

主役はコンテンツです。装飾は文脈を補助するためにだけ使います。
視線誘導のための強い色や大きな演出は、本当に必要な箇所に限定します。

### 2. Quiet Structure

画面は余白、細い境界線、淡い背景差で構成します。
カードや区切り線は使ってよいですが、情報を過剰に箱で囲わないようにします。

### 3. Small but Clear Hierarchy

見出し、本文、補足の差は大きくしすぎません。
このサイトでは「強いジャンプ」より「自然な段差」の方が合っています。

### 4. Native Reading Experience

日本語本文が多いため、文字組みと行間を優先します。
一画面での情報密度は上げすぎず、長文でも追える流れを保ちます。

## Visual Language

### Color

- ベースはグレースケールで構成する
- 背景は `main-bg`、補助面は `sub-bg` を使う
- テキストは `main-text` と `sub-text` を基本にする
- アクセントカラーは常用しない
- 赤、オレンジ、緑は意味を持つ箇所だけに使う
- ダークモードでもコントラストの関係は維持する

### Token Usage

`src/styles.css` では、見た目を直接値で決めるのではなく、テーマトークンを経由して揃えます。
新しい UI を追加するときも、まず既存トークンで表現できるかを確認します。

- 色は `bg-main-bg`、`bg-sub-bg`、`text-main-text`、`text-sub-text` を優先する
- ダークモード対応が必要な場合は `dark:bg-dark-main-bg` や `dark:text-dark-sub-text` を対にして書く
- 枠線は `border-gray-6` や `dark:border-dark-gray-6` のように、中間のグレーを基本にする
- 装飾のためだけに `gray-8` 以上の強い色を広い面へ使いすぎない
- 新しい色名を増やす前に、既存の gray scale と accent color で目的を満たせるか確認する

よく使う組み合わせ:

- ページ背景: `bg-main-bg dark:bg-dark-main-bg`
- 補助面: `bg-sub-bg dark:bg-dark-sub-bg`
- 主本文: `text-main-text dark:text-dark-main-text`
- 補足テキスト: `text-sub-text dark:text-dark-sub-text`
- 控えめな境界線: `border border-gray-6 dark:border-dark-gray-6`

### Typography

- 既定は `font-ud` を軸にする
- 本文は `text-base` を基準に、必要なら `text-sm` まで
- 補足情報やメタ情報は `text-xs` か `text-xxs`
- 大見出しを乱用しない
- 1行の長さは長くしすぎない。本文はおおむね `80ch` 以内を守る

使い分けの目安:

- 本文: `text-base`
- 短い補足文やナビゲーション: `text-sm`
- 日付やメタ情報: `text-xs`
- キャプションに近いごく小さな補足: `text-xxs`

フォント運用:

- アプリ全体の既定はルートで設定する
- 個別コンポーネントでフォントを切り替える場合も、`font-ud` か `font-sans` の範囲に留める
- 強調はフォント変更より、ウェイトと余白で付ける

### Spacing

- 余白は `8 / 16 / 24 / 40 / 64` を基本単位にする
- 近い要素は `8` または `16`
- セクション内のまとまりは `24`
- セクション同士は `40` 以上で離す
- レイアウトのリズムは余白で作り、不要な罫線に頼りすぎない

Tailwind では次を基準に使う:

- 極小の差分: `gap-4`
- 近い要素の間: `gap-8`, `mt-8`, `mb-8`
- 標準的な余白: `gap-16`, `px-16`, `py-16`
- コンポーネント内部の余白: `p-24`
- セクション間の余白: `py-40`, `space-y-40`
- 大きなレイアウト余白: `py-64`

避けたいこと:

- `17px` のような単発値を増やすこと
- コンポーネントごとに独自リズムを持たせること
- 余白不足をボーダーや背景色でごまかすこと

### Radius and Borders

- 半径は小さめを基本にする
- 通常の面は `rounded-6` か `rounded-8`
- 円形アバターのような意味のある要素だけ `rounded-full`
- 枠線は主張させず、背景差とセットで使う

目安:

- テキスト主体のカード: `rounded-6`
- 少しやわらかく見せたい面: `rounded-8`
- 大きなヒーロー風の過剰な角丸は持ち込まない
- ボーダーだけで成立しないときは、先に背景差を足す

## Layout Rules

- メインカラムは細めに保つ
- 既存の `max-w-2xl` を標準とする
- ページ全体の左右余白は `px-24` を基準に、広い画面では `sm:px-40`
- ヘッダーは補助的な存在として扱い、本文より目立たせない
- 新規ページでも、まずは 1 カラムで成立する構成を優先する

## Component Guidance

### Header

- 常にナビゲーションの起点として機能する
- 情報量は最小限にする
- パンくずの代わりに、今いるセクションだけを静かに示す

### Profile

- トップページの導入として、人と内容の両方が伝わる構成を保つ
- 自己紹介文、導線、画像の順序は崩しすぎない

### Timeline

- 時系列の可読性を最優先する
- 装飾より、日付と要約の関係がすぐ読めることを重視する
- カード表現は使ってよいが、影より境界線と背景差を優先する

### Markdown Content

- ブログや自己紹介ページは、読み物としての整った組版を優先する
- 埋め込み UI を増やしすぎない
- 本文中のリンク、コード、リストは素朴で明瞭な見え方にする

## Interaction Rules

- アニメーションは最小限にする
- まずは静止状態の美しさを優先する
- ホバーやフォーカスは、装飾ではなく状態変化として明確にする
- `focus-visible` は削らない
- sticky ヘッダーや blur のような効果は、可読性を損なわない範囲で使う

## Accessibility

- 色だけで意味を伝えない
- フォーカスリングは必ず見える状態を保つ
- 画像には用途に応じた `alt` を入れる
- 見出しレベルは見た目ではなく文書構造で決める
- 小さい文字を使うときほどコントラストを意識する

## Implementation Notes

- 新しい色や余白の値を直接書く前に、既存トークンで表現できるか確認する
- Tailwind のユーティリティは、既存の命名とスケールに揃える
- 新しい見た目が必要なら、まず `src/styles.css` のテーマと整合する形で追加する
- 一時的な例外値を増やしすぎない
- ページごとの個性は、色の追加よりコンテンツ構成で出す

実装時の優先順位:

1. 既存ユーティリティで組む
2. 既存トークンを使って組む
3. 必要なら `src/styles.css` にトークンを追加する
4. 最後の手段として、その場限りの値を使う

トークン追加の判断基準:

- 複数箇所で再利用されるか
- ライトとダークの両方で意味を持つか
- 既存スケールの間に自然に収まるか
- 名前が用途ではなく役割を表せているか

## When Unsure

迷ったときは次の順で判断します。

1. 読みやすいか
2. 既存ページと並べて違和感がないか
3. 装飾が情報より前に出ていないか
4. ダークモードとキーボード操作でも成立するか

## Non-goals

- マーケティングサイトのような強いヒーロー演出
- 過剰なグラデーションや大きな影
- 多色でにぎやかな UI
- コンテンツを圧迫する密なダッシュボード表現
